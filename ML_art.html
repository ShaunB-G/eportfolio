    Unit08: Gradient Cost Function

    Learning Outcomes:

  • Understand the applicability and challenges associated with different datasets for the use of machine learning algorithms.
    • Download Notebook

    Experiment 1: Default values (iterations=100, learning_rate=0.08)

    Experiment 2: Higher iterations (iterations=500, learning_rate=0.08)

    Increasing the iterations saw a vast improvement in the reduction of the cost value. With this data did not noticabily increase the processing time. However, it might lead to longer computation times for larger datasets. Generally helps in achieving a more accurate and refined convergence. However, it might lead to longer computation times and diminishing returns beyond a certain point.

    Experiment 3: Lower Iterations (iterations=50, learning_rate=0.08)

    Decreasing the iterations saw a vast increase of the cost value by the last iteration. Having already completed a experiment with more iterations it is obvious that this number of iterations is no where near enough and increasing the iterations will vastly reduce the cost without any negative outcomes.

    Experiment 4: Higher learning rate (iterations=100, learning_rate=0.2)

    A small increase in the learning rate caused the cost to increase with each itteration this suggests that the minimal was missed. Can lead to faster convergence but risks overshooting the minimum/oscillating around it. Choosing a learning rate that is too high may prevent convergence altogether.

    Experiment 5: Lower learning rate (iterations=100, learning_rate=0.01)

    The algorithm required more iterations to reduce the cost to the same level as when a higher learning rate was used. Slowing down the convergence can provide stability, especially in cases where a higher learning rate would cause overshooting. However, it might require significantly more iterations to find the minimal.

    Key takeaways:

  • Finding the Right Balance: The optimal combination of iterations and learning rate depends on the specific characteristics of the dataset and the processing power/time availble. It may involves experimentation and fine-tuning to strike the right balance.
  • Trade-offs: There is a trade-off between convergence speed and stability. Increasing one may come at the expense of the other, and finding the right compromise is crucial for effective gradient descent.
  • Monitoring Convergence: Regularly monitoring the cost values and convergence behavior during training helps in identifying the effectiveness of the chosen hyperparameter values and making adjustments as needed.

    Unit 11 e-Portfolio Activity: Model Performance Measurement

    This task involved running the model_Performance_Measurement.ipnyb file and changing different parameters to observe its impact on AUC and R2 error.

    Logistic Regression Model

    Altering C value

    As C increases (from 0.1 to 100), both AUC and R2 also increase. This suggests that a higher value of C, which corresponds to a lower regularization strength, leads to a better fit for this particular model on the given dataset.

    C AUC R2
    0.1 0.9904731660844712 0.7465862178469356
    1.0 0.9942838996506828 0.8189901556049539
    10.0 0.9965068275643061 0.8189901556049539
    100 0.9968243886948237 0.8189901556049539

    Altering Solver

    Comparing 'liblinear' and 'lbfgs' solvers, 'liblinear' resulted in a higher AUC and R2. This indicates that, for this specific problem and dataset, the 'liblinear' solver performs better.

    Solver AUC R2
    liblinear 0.9942838996506828 0.8189901556049539
    lbfgs 0.9892029215624007 0.7465862178469356

    Altering Random State

    Changing the random state did not have a significant impact on AUC and R2. All three random states (0, 42, and 99) resulted in the same AUC and R2 values. This suggests that the model's performance is relatively stable across different random splits of the data.

    Random State AUC R2
    0 0.9942838996506828 0.8189901556049539
    42 0.9942838996506828 0.8189901556049539
    99 0.9942838996506828 0.8189901556049539

    In summary, for this logistic regression model on the breast cancer dataset:

  • Increasing the regularisation strength (decreasing C) generally leads to a decrease in model performance.
  • The choice of solver can impact performance, with 'liblinear' performing better in this case.
  • Random state changes did not affect the model's performance significantly, indicating stability in the results across different random splits.
